## 2. Findings ordered by impact

### CRITICAL — Confirmed

**C1. Hook rewrites the producer side of pipelines → downstream stages receive compressed data; silently wrong results**
- Severity: critical · Confidence: Confirmed [R][V]
- `src/hook/rewrite.ts:269-271` — only segments with `precededBy === "|"` (RHS) are skipped; a segment whose stdout feeds a pipe is still rewritten. ADR 0007 follow-up #1 ("don't compress a segment whose stdout flows into a downstream stage") is unimplemented.
- Trigger: any hooked `cmd-with-handler | filter`, e.g. `git diff | grep -c '^+'`.
- Repro: plain pipeline → `3`; rewritten `tk git diff | grep -c '^+'` → `0`.
- Impact: data corruption (not loss) delivered to the agent under the primary live delivery path (Claude Code hook).
- Fix: track `followedBy` per segment in `rewriteCommand`; skip rewriting any segment followed by `|`.
- Regression test: `rewriteCommand("git diff | grep -c '^+'")` → decision `pass` (or head untouched); end-to-end count parity in `tests/unit/hook/`.
- Effect: loses hook compression on pipe heads (correct per ADR 0007 — pipe tails were already measured as not worth compressing).

**C3. Shell `test` builtin hijacked: `test -f x && …` breaks with exit 127**
- Severity: critical · Confidence: Confirmed [R][V]
- `src/handlers/system/testRunner.ts:95-107` — `matches: program === "test" && args.length > 0`; `execute` spawns `args[0]` as the program, so `tk test -f package.json` runs program `-f` → 127. Hook eligibility includes it: `rewriteCommand("test -f package.json && echo yes")` → rewrite.

**C7. Shim fork-bomb (known D3) still open: wrappers are not self-guarding**
- Severity: high(env-conditional)→critical impact · Confidence: Confirmed [V]
