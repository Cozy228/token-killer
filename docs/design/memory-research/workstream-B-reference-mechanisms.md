# Workstream B — Reference Tool Mechanisms

> **Framing law (per go-prompt Rule 3 + Workstream-B brief).** This document extracts *mechanisms only*.
> No entry says "ctx should copy X." Every notable mechanism is phrased **"X demonstrates mechanism Y;
> ctx relevance = Z"** where Z ∈ { **adopt-worthy** · **translate-with-changes** · **decline-violates-invariant** ·
> **irrelevant** }. Every claim is labeled `from-reference` (with clone `path:line`) or `inferred`. Verdicts are
> judged against ctx's five hard invariants and the Decision-2 failure priorities, not against how good the tool is.
>
> **ctx invariants used as the pass/fail gate.** (1) no LLM / no embeddings / no network at **write OR serve** time;
> (2) one local **SQLite + FTS5** store per project (lexical FTS is the retrieval engine); (3) **index-not-copy** except
> memory & concepts (authored, stored in full); (4) **provenance per fact** (source + authority + time); (5) conflicts
> are **surfaced**, never averaged / merged / auto-resolved.
>
> **Decisions each profile feeds** (from the go-prompt): (2) failure priorities · (3) write policy manual-vs-automatic ·
> (4) lifecycle / supersede / delete / review · (5) anchor freshness on target change · (6) retrieval / ranking signals —
> which are dangerous · (7) push-digest contents + exclusions · (8) host import what/never + echo prevention (incl.
> paraphrase + needs-review default) · (9) human-vs-agent controls.

## (ii) Per-tool profiles

Profiles are compact; the matrix above carries the at-a-glance shape. Each ends with the **Decisions** it informs.

### A. Dedicated agent-memory systems

#### mem0 — informs (2)(3)(4)(6)(7)(8)
- **Memory unit** `from-ref`: one short NL fact string + embedding + metadata (`user/agent/run/actor/role/hash/created_at/updated_at/expiration_date`); parallel deduped-entity collection with `linked_memory_ids` (retrieval boost, not a relation graph).
- **Write trigger** `from-ref main.py:868-1158`: automatic per `add()` (LLM fact-extraction, `infer=True`). **Evidence gate is deterministic — MD5 exact-hash dedup** (`main.py:957-976`), no semantic-dup rejection at write. **Key finding**: the classic MemGPT `ADD/UPDATE/DELETE/NONE` LLM decision prompt still exists (`prompts.py:176-460`) but is **dead code** — only caller is a unit test; live pipeline is **additive-only**, every row is `"event":"ADD"` (`main.py:1147-1150`). `UPDATE`/`DELETE` are **explicit human/dev API only** (`main.py:1767-1829`).
- **Echo guard `from-ref`**: extraction prompt says *"Use [Existing Memories] ONLY for deduplication and linking — do NOT extract new memories from Existing Memories"* (`prompts.py:511`) — a built-in "don't re-mine your own context" rule.
- **Provenance**: source/actor/time yes; **no authority/confidence**, no evidence pointer beyond hash.
- **Lifecycle**: no auto-staleness in OSS; hard TTL via `expiration_date`; append-only `history` ledger (event/old/new/actor/time).
- **ctx verdicts**: hash-exact-dedup-before-insert = **[adopt-worthy]** (deterministic, invariant-1 OK if over raw text). "Existing-memories-shown-for-dedup-only, never re-extract" = **[translate-with-changes]** — encode as a hard rule in ctx's write path (ctx has no LLM prompt to carry it). Additive-only-never-resolve (V3) = **[decline-violates-invariant]** — worse than auto-merge for ctx: it *never* surfaces conflict (invariant 5). Legacy ADD/UPDATE/DELETE prompt = **[decline]** (LLM at write + auto-resolve). Embedding+BM25+entity-boost fusion = **[irrelevant]** (embeddings); the *entity-linking boost idea* has a lexical co-occurrence analogue. `expiration_date` TTL + append-only history ledger = **[adopt-worthy]** (matches invariant 4/5 shape).

#### letta (MemGPT) — informs (3)(4)(5)(6)(8)(9)
- **Memory unit** `from-ref block.py:13-77`: a **Block** `{label,value,limit,read_only,tags,created_by_id,last_updated_by_id}`. Three tiers: **core** (Blocks rendered directly in context), **archival** (`Passage` = text+embedding, searchable, out of context), **recall** (raw transcript).
- **Write trigger** `from-ref base.py`: agent self-edits via tool calls — `core_memory_append` (no dedup, `:246-260`), `core_memory_replace` (**exact-match-or-raise**, `:263-280`), `rethink_memory` (full-block LLM rewrite "integrate…not outdated", `:283-300`), unified `memory(create/str_replace/insert/delete/rename)` (`:10-68`). **Automatic path**: `SleeptimeMultiAgent` — a *separate* LLM agent every N messages idly rewrites the primary's blocks (`sleeptime_multi_agent.py:23-70`).
- **Substrate**: SQL + **real git commit per edit** (`git_operations.py:351-424`, author "Letta Agent", Redis-locked) → free diff/blame/revert; archival uses pgvector.
- **Scope** `from-ref blocks_agents.py:7-10`: Block↔agent **many-to-many** (deliberately shared live across agents).
- **Lifecycle**: no auto-staleness; conflict = agent judgment at rewrite; `read_only` = only structural veto; **no pending-review queue** for self-edits.
- **ctx verdicts**: `core_memory_replace` exact-match-or-raise = **[adopt-worthy]** (cheap deterministic no-op guard). `read_only` block flag = **[adopt-worthy]** (structural human-vs-agent veto). Git-commit-per-edit = **[translate-with-changes]** (shape informs ctx's provenance ledger; ctx mandates SQLite not a git subprocess). M:N block↔consumer sharing = **[translate-with-changes]** (validates one stored memory linked to N consumers). `rethink_memory` + `SleeptimeMultiAgent` = **[decline-violates-invariant]** (LLM auto-resolves conflicts unaudited; two-writer drift). Archival embedding search = **[irrelevant]**.

#### zep + graphiti (bi-temporal engine) — informs (2)(4)(5)(6)(8)(9) — **highest-value lifecycle reference**
- **Memory unit** `from-ref edges.py:49-72`: an `EntityEdge` = one fact `{fact, fact_embedding, episodes[], valid_at, invalid_at, expired_at}`. **Nodes are never invalidated — only edges/facts carry temporal validity** (grep `invalid_at` in nodes.py = 0 hits).
- **Write trigger**: auto (LLM extract), **sequential per `group_id`** to avoid write races (`graphiti_mcp_server.py:359-360`). Conflict handled in two split steps: LLM returns *separate* `duplicate_facts` vs `contradicted_facts` lists (`edge_operations.py:726-776`) — identity and contradiction are different questions; then a small LLM call sets `valid_at/invalid_at`. **The actual invalidation write is deterministic date arithmetic, not an LLM decision** — `resolve_edge_contradictions` compares intervals and stamps the older edge `invalid_at=new.valid_at, expired_at=now()` (`edge_operations.py:538-573`); insertion order never overrides temporal truth — a *newer-than-the-new-fact* candidate makes the new edge born already-expired (`:820-839`). **Both rows persist forever; one is flagged invalid.**
- **Dedup** `from-ref dedup_helpers.py:1-296`: fully deterministic pre-pass **before any LLM** — exact normalized-name, then **MinHash(32)/LSH(band=4)+Jaccard≥0.9 gated by a Shannon-entropy check** (short/low-entropy names defer to escalation, not fuzzy-trust) — *this is the same shape as ctx's own `dedup.ts` entropy floor + Jaccard.* Only genuinely-unresolved names escalate (graphiti escalates to LLM; ctx would escalate to a human queue). Caveat: candidate *retrieval* for that pass uses embedding cosine.
- **Ranking**: BM25 + cosine fused by **RRF** (deterministic); **`node_distance_reranker`** graph-walk (deterministic); MMR diversity. **Danger `from-ref graphiti.py:1527-1533`**: default `search_filter=None` means **expired facts are NOT excluded by default** — stale ranks alongside live unless the caller opts in.
- **Provenance**: genuine **bi-temporal** (2 time axes) + `episodes[]` source pointers; no authority/confidence.
- **Human controls**: hard-delete-by-UUID only; **no soft "human marks invalid" lever, no review-before-write queue** — the LLM's contradiction call resolves + commits unsupervised.
- **ctx verdicts**: bi-temporal `valid_at/invalid_at/created_at/expired_at` never-delete = **[adopt-worthy]** — most directly transferable model for Decision 4 (supersede) + 5 (freshness); the invalidation *math* needs zero LLM/embeddings once two facts + timestamps are known. Split "same-claim vs contradicts" = **[translate-with-changes]** (keep the two-question shape; replace LLM with same-anchor+FTS-near-dup for identity, and **human-only** for contradiction under invariant 5). Entropy-gated MinHash/LSH dedup with escalate-when-uncertain = **[adopt-worthy]** (zero-LLM; escalate to human queue not LLM). RRF + node-distance graph-walk = **[adopt-worthy]**; cosine half = **[irrelevant]**. **Auto-stamp-invalid-on-detection with no review = [decline-violates-invariant]** (invariant 5). **Default-off temporal filtering = [decline as designed]** — the concrete "dangerous ranking signal" ctx must invert: invalidated claims filtered/down-ranked **by default**, never opt-in. Hard-delete-only (no soft human invalidate) = **[translate-with-changes]** (a gap to avoid).

#### cognee — informs (2)(3)(4)(6)(8)(9)
- **Memory unit** `from-ref DataPoint.py:27-70`: `DataPoint` universal graph-node base.
- **Write trigger**: automatic `cognify()` ECL pipeline (classify→chunk→**LLM extract**→**LLM summarize**→load), **zero evidence gate**; LLM_API_KEY load-bearing → **[decline-violates-invariant]** for the population path.

- **ctx verdicts**: identity-by-construction id = **[adopt-worthy]** (best first line of defense, strictly simpler than fuzzy). Rich provenance stamp = **[adopt-worthy]** (fits invariant 4 better than graphiti's thin episode list). `on_conflict_do_nothing` first-run-wins = **[adopt-worthy]** (safe rollback). Human-feedback→EMA→per-fact weight = **[adopt-worthy]** (deterministic human ranking influence). BM25-first-class = **[adopt-worthy]** (corroborates FTS5). `MERGE…SET n+=props` silent overwrite = **[decline-violates-invariant]** (invariant 5 "don't do this"). pipeline-index `topological_rank` = **[decline]** dangerous signal. cognify LLM/embedding pipeline + truth-subspace = **[decline-violates-invariant]**.

### B. Local file/note memory

#### basic-memory — informs (3)(4)(5)(6) — **closest design cousin**

Move-conflict→raise = **[adopt-worthy]** (surface-don't-auto-resolve at file layer). sqlite-vec/embedding layer = **[decline-violates-invariant]** + **cautionary**: even the lexical-first cousin drifted to embeddings under pressure → ctx should treat "stay lexical" as a *standing* decision.

- **The invariant-5 review queue is near-novel — exactly one deterministic precedent.** `from-reference`, across every *graph/vector* memory system (mem0 / letta / zep-graphiti / cognee / a-mem / rohitg) and every *LLM-rewrite* coding tool (cline / continue / gemini-cli Path A): **none surfaces a conflict to a human before committing** — all merge, pile, rewrite, or auto-invalidate.
