## 1 Executive Verdict

- [from-doc][inferred] ctx memory should stay a narrow project-context fact layer: short, provenanced experience notes that do not fit better as code, docs, decisions, requirements, history, or derived concepts. The product definition is broader than memory and centers local Acquire/Verify/Serve/Refresh across all context sources, so memory is one content type in that base, not the product itself (`CONTEXA-DESIGN.md:21-29`, `CONTEXA-DESIGN.md:39-52`).
- [from-code][inferred] The current architecture is directionally right: memory is a `memory` entity plus a side table, anchors, claims, links, and FTS rows. `remember()` resolves anchors all-or-nothing, writes `anchoredTo` claims/links, and keeps superseded rows by status change rather than deletion (`packages/core/src/memory/remember.ts:165-236`).
- [inferred] ctx memory is not a chatbot memory, rule-authoring system, skill-learning layer, transcript compressor, semantic wiki, or host-instruction generator. Those products optimize adaptive behavior; ctx optimizes faithful, local, reviewable project facts.
- [from-doc][inferred] The top failure to make impossible is privacy/egress breach because the hard invariants forbid LLMs, embeddings, and network at write/serve time. Within that boundary, the design should optimize first against false/stale memory, second against host echo/unreviewed imports, and third against irrelevant push/context pollution (`docs/build/MEMORY-RESEARCH-GOAL-PROMPT.md:40-41`, `docs/design/memory-research/workstream-D-biology.md:48-58`).
- [from-code][inferred] The largest current risk is that lifecycle status is not a pull-serving filter. `rankGotchas()` lists only active memory for push, but selection seeds/subgraph/ranking only check published-generation visibility, so retired or superseded memory can still be recalled by `context()`/`search()` unless the query path misses it (`packages/core/src/push/rank.ts:73-84`, `packages/core/src/select/visibility.ts:44-52`, `packages/core/src/select/seeds.ts:75-82`).
- [from-code][inferred] The second largest current risk is host import. Claude imports are `authority:"inferred"` but `status:"active"`, while the refresh engine never calls the memory adapter because `dirtyCheck()` is always clean; the CLI nevertheless says local host memory is imported automatically on cold-path sync (`packages/core/src/memory/claudeImporter.ts:214-224`, `packages/core/src/memory/adapter.ts:17-23`, `packages/core/src/ingest/refresh.ts:122-125`, `packages/cli/src/cli.ts:239-245`).
- [from-code][from-reference][inferred] The top three design changes are: default all host-imported or agent-proposed memory to `needs-review`; add anchor-freshness invalidation that records reason-classed `stale-suspect` conflicts and excludes stale memory from push/default pull; add a deterministic prewrite/review reconciliation vocabulary so duplicates, supersedes, and unsafe imports become review items rather than active facts (`docs/design/memory-research/workstream-B-reference-mechanisms.md:273-294`, `docs/design/memory-research/workstream-E-evaluation.md:426-431`).
- [from-reference][inferred] Adopt small deterministic mechanisms from references only when translated to ctx: bitemporal validity as schema discipline, checksum/hash freshness, RRF/anchor-proximity ranking, MMR-style duplicate suppression, inbox review, and exact/id-level evaluation. Decline LLM selectors, embeddings, automatic rewrites, popularity ranking, remote memory surfaces, and read-own-summary rewrite loops (`docs/design/memory-research/workstream-B-reference-mechanisms.md:298-311`, `docs/design/memory-research/workstream-C-papers.md:148-162`).

## 4 Recommended Memory Model

- [from-code][inferred] **Memory unit.** A memory is `mem:<ulid>` plus a <=240 char gist, optional detail, origin, authority, status, optional session_ref, anchors, and backing claims/links. The store is source of truth for memory gist/detail; other local content remains index-not-copy (`packages/core/src/store/migrations/001-init.sql:54-75`, `packages/core/src/store/store.ts:629-643`).
- [from-doc][inferred] **Authority.** Manual `remember()` and human notes default `confirmed`; host imports and machine proposals default `inferred` plus `needs-review`. Authority is evidence provenance, not a rank-only hint. It upgrades only by explicit human confirmation (`packages/core/src/memory/remember.ts:201-208`, `packages/core/src/memory/claudeImporter.ts:214-224`).
- [from-code][inferred] **Provenance.** Every anchor, supersede, duplicate candidate, stale reason, and lifecycle transition should have a claim/event with carrier, locus where available, method, authority, and timestamp. Current anchor/supersede claims are good; lifecycle transitions need an equivalent audit trail (`packages/core/src/memory/remember.ts:211-253`, `packages/core/src/store/store.ts:79-93`).
- [from-code][inferred] **Anchors.** Anchors are explicit retrieval cues to entity ids. Write-time resolution remains all-or-nothing; file anchors may auto-create real project-relative file entities; unresolved anchors produce candidates and no write (`packages/core/src/memory/remember.ts:99-132`, `packages/core/src/memory/remember.ts:165-178`).
- [from-reference][inferred] **Lifecycle.** Active means current and eligible; needs-review means quarantined but inspectable; superseded means validity ended by a newer fact; retired means intentionally hidden. Forgetting means accessibility/rank suppression, not deletion (`docs/design/memory-research/workstream-D-biology.md:11-20`).
- [from-doc][inferred] **Freshness.** Anchor freshness is structural: target exists, target content hash/fingerprint still matches, and target identity is not ambiguous. Drift sets `needs-review`, writes a `stale-suspect` conflict with reason class, and excludes from push/default pull (`CONTEXA-IMPL.md:196-205`, `CONTEXA-IMPL.md:540-544`).
- [from-code][from-reference][inferred] **Ranking.** Use FTS5 and the shared graph as today, but add eligibility first. Rank eligible memory by lexical relevance, anchor proximity, authority, explicit pin, freshness, and duplicate diversity. Recency is weak; served_count is review telemetry only. Decline embeddings, LLM importance, and popularity rank (`packages/core/src/select/rank.ts:42-93`, `docs/design/memory-research/workstream-C-papers.md:148-162`).
- [from-doc][inferred] **Push eligibility.** Push is not a review queue. It includes a fixed header and only high-confidence active fresh gotchas/conventions. Hard-exclude retired, superseded, needs-review, stale-anchor, unconfirmed host import, unresolved-anchor, sameAsCandidate-unresolved, and echo-risk. Pin orders eligible items; veto excludes everything (`docs/build/MEMORY-RESEARCH-GOAL-PROMPT.md:88-92`, `packages/core/src/push/block.ts:85-120`).
- [from-code][from-reference][inferred] **Host imports.** Importers are evidence collectors, not trust gates. Claude/Codex/Copilot host memory or instruction files must be project-scoped, local, provenance-marked, sentinel-stripped, checked against ctx-origin pushed gists, and inserted as `needs-review` unless explicitly human-authored and trusted. Never write ctx content into host auto-memory directories (`packages/core/src/memory/claudeImporter.ts:152-267`, `docs/design/memory-research/workstream-B-reference-mechanisms.md:291-311`).
- [from-doc][inferred] **Human review.** `ctx guide` should be read-only and evidence-first: review queue, stale-reference list, duplicate candidates, push pin/veto state, and evidence drawer. Mutations remain CLI/library commands until a maintainer explicitly chooses otherwise (`CONTEXA-DESIGN.md:181-187`, `docs/design/memory-research/workstream-E-evaluation.md:365-431`).
- [from-code][inferred] **Agent retrieval.** Agents get three tools: `context`, `search`, `remember`. Tool output must carry handles, freshness/status labels, conflicts when relevant, and drill-down detail. Agents can propose/write memory but should not confirm host imports or resolve lifecycle without an explicit policy (`packages/cli/src/mcp.ts:37-93`).
